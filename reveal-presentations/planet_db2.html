## Databases ### LO: apply our knowledge of databases to build an application
### Relational Databases Last week we imported some data, added attachments and worked with forms This week we will look at queries and reports
### First import all the data - Open access (start a new empty database) - Import the planet-data and moon-data files - When you import the planet data set the primary-key to be the 'planet' field - Open up the new tables and make sure it looks OK - Save your database.
### Create a new relationship between moons and planets - Under the 'Database tools' tab click the relationships button - Add each table to the relationship viewer - Use the edit relationships button to create a new relationship - Choose the moon-data/orbits for the left hand side and planet-data/planet for the right - Save your database.
### Creating queries over both tables - Under the create tab click query design Lets make a query that lists only those planets which have moons - At the bottom of the page set the table in the first column to planets-data, set the second column to table: moons-data, field: moon - When you run the query you will see a line for every moon which is linked to a planet - Now try the queries in the example-queries file on moodle.
### Making a report Just like forms can provide a nicer interface for data-entry, reports can make our query results loook much nicer. - Use the create tab to make a report from your query.
