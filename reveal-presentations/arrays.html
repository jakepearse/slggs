Where are variables stored?

When we declare a variable, we are reserving a space in RAM 

int x;

A Java int is a 32bit number, so we have reserved 32bit in RAM

x = 4294967295;

Our value is stored in the space we reserved, we have no real idea where exactly in RAM that is.

Multiple Values

If we want to store a lot of related data in RAM, we could just start declaring loads of variables 

int dataA;
int dataB;
int dataC;
...

This approach very quickly becomes unhelpful.

There is no easy way to establish that these values are related to each other, except that they have similar identifiers.

Structured Data

When we need to represent the relationship between values we need a Data Structure, The simplest (maybe most important) data structure is the array.

In Java we declare an array using square brackets.

int[] myIntArray;

This only reserves the point that our array will start at in RAM.

We also need to tell the compiler how many elements our array is going to contain.

myIntArray = new int[3];

Writing to an Array

Of course we can only assign values of the correct type to our array elements.

The first element of any array is at index zero [0]

myIntArray[0] = 33;

The value 33 is now stored at the 0th element; Index 0

If we try to write to (or read from) an element that is equal to the length of array, it will fail.

ArrayIndexOutOfBoundsException

Arrays have a property called length which is the number of elements the array can hold

Reading from an array

We can get the value from any index simply. 

myIntArray[2];

Returns the value stored at index 2

The following code will always fail

myIntArray[myIntArray.length];

Can anyone see why?

myIntArray[myIntArray.length - 1];
