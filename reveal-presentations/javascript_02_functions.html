Literals

Variables identify a value, that value can change over time.

A literal simply expresses the value itself. Literals don't stand for something else, they literally are that value.

Variables

We can use variables to 'grasp' our values

var name;

Now we have a variable labelled name we have not given it a value so it has the special value undefined

name = "Joelle Rollo-Koster";

The single equals operator is used for assignment

The value returned by the expression on the right is assigned to the variable on the left

Not a number

If we try to form an expression using a string and a number, we might get the special value

NaN

This is a particular oddity of JavaScript, although it's not a number it partially behaves like one!

"tea" / 2;
<·NaN

For the most part, if you see NaN cropping up, it means you have tried to perform a mathematical operation on a string by mistake.

Boolean values

In Boolean logic there are only two possible literal values

true;

false;

This type of value would be returned from a comparison expression

27 <= 5;

2 * 5 === 10;

Strings

We have already seen the string literal.

As well as being a literal value strings are objects

This means they have properties and methods.

Try:

"one".length;
